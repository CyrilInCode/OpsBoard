# SonarQube Setup

This project supports two quality workflows:

- Local SonarQube Server with Docker Compose, useful for screenshots and demos.
- SonarQube Cloud in GitHub Actions, useful for pull requests and the final repository.

## Local SonarQube

Start the local quality stack:

```bash
docker compose --profile quality up -d sonarqube
```

Open http://localhost:9000 and sign in with:

- Login: `admin`
- Password: `admin`

SonarQube asks for a new admin password on first login. Then create a local project:

- Project key: `opsboard-devops`
- Display name: `OpsBoard DevOps`
- Analysis method: locally
- Token name: `opsboard-local`

Run the project checks before scanning:

```bash
mvn -B -f backend/pom.xml verify
cd frontend
npm ci
npm run test:coverage
cd ..
```

Then launch the scan with the token generated by SonarQube:

```bash
docker run --rm \
  --network projet_default \
  -e SONAR_HOST_URL=http://sonarqube:9000 \
  -e SONAR_TOKEN=replace_with_local_token \
  -v "$PWD:/usr/src" \
  sonarsource/sonar-scanner-cli \
  -Dsonar.projectKey=opsboard-devops
```

The dashboard is then available at:

```text
http://localhost:9000/dashboard?id=opsboard-devops
```

## GitHub And SonarQube Cloud

Create a project on SonarQube Cloud and bind it to the GitHub repository.

In GitHub, go to:

```text
Repository > Settings > Secrets and variables > Actions
```

Add this repository secret:

```text
SONAR_TOKEN
```

Use a token generated from SonarQube Cloud. Keep it secret; never commit it.

Add these repository variables:

```text
SONAR_PROJECT_KEY
SONAR_ORGANIZATION
```

The project key and organization key are visible in the SonarQube Cloud project settings.
They must be created as **Variables**, not as secrets. If they are missing, GitHub expands them to empty strings and the scanner receives invalid arguments such as `-Dsonar.projectKey=`.

The GitHub Actions workflow runs:

```bash
mvn -B -f backend/pom.xml verify
cd frontend && npm ci && npm run test:coverage && npm run build
```

Then it launches `SonarSource/sonarqube-scan-action@v7` with:

```text
sonar.projectKey=${{ vars.SONAR_PROJECT_KEY }}
sonar.organization=${{ vars.SONAR_ORGANIZATION }}
sonar.qualitygate.wait=true
```

If the quality gate fails, the GitHub Actions job fails too.
